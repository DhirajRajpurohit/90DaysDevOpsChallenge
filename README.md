# 🚀 #90DaysOfDevOps

Welcome to my **90-day DevOps learning challenge** — a daily, hands-on journey from networking fundamentals to cloud infrastructure, automation, and beyond. This repo is my public log: what I learn, what I build, what breaks, and how I fix it.

---

## 🎯 Goal

I'm spending the next 90 days building real, practical DevOps skills — not just reading theory, but actually doing the work: running commands, breaking things in a sandbox, and documenting the fix.

By the end of 90 days, I want to be able to:
- Confidently troubleshoot networking issues from the OS level up through the cloud
- Design and deploy secure, well-structured cloud infrastructure (Azure-first, with AWS/GCP concepts where relevant)
- Understand and use core DevOps tooling — Linux, Git, CI/CD, containers, IaC, monitoring
- Move from "I followed a tutorial" to "I can debug this myself in a real incident"

This challenge is as much about **consistency** as it is about content — showing up daily, even on the days where the only progress is one small command finally making sense.

## 🧠 What I'm Covering (Roadmap)

This isn't fixed in stone — it'll adjust as I go — but the rough plan:

- **Networking fundamentals** — OSI/TCP-IP, DNS, ports, routing, diagnostics
- **Cloud networking (Azure)** — VNets, NSGs, Load Balancers, Application Gateway, Firewall, VPN Gateway, ExpressRoute
- **Linux fundamentals** — file systems, permissions, processes, shell scripting
- **Git & version control** — branching strategies, collaboration workflows
- **CI/CD** — pipelines, automated testing, deployment strategies
- **Containers & orchestration** — Docker, Kubernetes basics
- **Infrastructure as Code** — Terraform / Bicep
- **Monitoring & observability** — logging, alerting, incident response

---

## ✅ How I Document Each Day

Every day follows roughly the same structure, so it's easy to revisit later:
1. **Theory** — concepts explained with a real, worked example, not just a definition
2. **Practical** — hands-on steps, either CLI commands or manual portal walkthroughs, with what to actually observe at each step
3. **Real incident scenarios** — "if X breaks, check Y" — the troubleshooting mindset, not just the happy path
4. **Quick reflection** — the one or two things worth remembering after the day
